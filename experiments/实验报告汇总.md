# USRP MIMO CSI 感知实验报告汇总

## 1. 实验目标

本项目目标是基于两台 USRP B210 构建一个可控的 2x2 MIMO CSI 采集链路，并验证从发射 OFDM probe 到接收端提取 CSI 的完整流程。

实验分为三个阶段：

```text
实验一：验证接收端可以在频谱上观察到发射端 OFDM 子载波能量
实验二：验证接收端可以稳定检测 probe frame，并确定合适的检测阈值
实验三：采集 5 秒 IQ 数据，离线提取 CSI，并分析 CSI 的幅度、相位和稳定性
```

对应目录：

```text
experiments/01_rx_subcarrier_power
experiments/02_frame_threshold_detection
experiments/03_5s_csi_extraction_analysis
```

## 2. 系统配置

实验使用两台 USRP B210：

```text
TX device: serial=326F493
RX device: serial=3271260
```

主要射频参数：

```text
center frequency: 1800 MHz
sample rate: 1 MS/s
RX antenna: TX/RX
TX antenna: TX/RX
RX gain: 40 dB
TX gain: 30 dB 或更高
```

OFDM probe 参数：

```text
FFT length: 64
CP length: 16
active subcarriers: -26..-1, 1..26
active carrier count: 52
subcarrier spacing: 15.625 kHz
active RF span: 1799.593750 MHz..1800.406250 MHz
probe rate: 1000 Hz
```

当前 frame 结构已经改为 WiFi-like training 形式：

```text
STF-like repeated short training
LTF-like repeated long training
TX0 pilot
TX1 pilot
guard
```

在 `rate=1e6`、`probe_rate=1000` 时：

```text
frame_len = 1000 samples
short training = 160 samples
long training = 160 samples
TX0 pilot = 80 samples
TX1 pilot = 80 samples
guard = 520 samples
```

## 3. 实验一：接收端观测发射端子载波功率

### 3.1 实验目的

实验一用于验证发射端启动后，接收端能够在频谱和瀑布图上观察到 OFDM probe 的能量分布。

这一阶段不要求提取 CSI，只验证 RF 链路是否打通：

```text
TX 是否真的在发射
RX 是否使用正确设备和端口
TX/RX 是否在同一中心频率和采样率
OFDM active subcarrier 频带是否能在 RX 频谱上看到
```

### 3.2 运行方式

先启动 RX 频谱 GUI：

```bash
bash start_rx_gui.sh
```

再启动 TX：

```bash
bash start_tx.sh
```

### 3.3 实验现象

RX GUI 中可以看到在中心频率附近出现一段宽带能量抬升，而不是单个尖峰。这是正常现象，因为 TX 发射的是 OFDM probe，能量分布在 52 个 active subcarriers 上。

证据文件：

```text
experiments/01_rx_subcarrier_power/evidence/spectrum_power_visible_01.png
experiments/01_rx_subcarrier_power/evidence/spectrum_power_visible_02.png
```

对应数据：

```text
experiments/01_rx_subcarrier_power/data/test_rx_1800
```

### 3.4 实验结论

接收端可以观测到发射端 OFDM probe 的子载波功率分布，说明：

```text
两台 B210 的基本 RF 链路可用
设备 serial、天线端口、频率、采样率配置正确
后续 frame detection 和 CSI extraction 具备实验基础
```

## 4. 实验二：阈值确定与 frame 检测

### 4.1 实验目的

实验二用于验证接收端能否稳定检测 probe frame 的起点，并确定合适的 frame detection threshold。

接收端实时 monitor 的处理流程为：

```text
短 IQ 缓存
-> preamble/training correlation
-> frame start detection
-> CFO 估计
-> pilot 切片
-> CSI 预览
```

### 4.2 运行方式

启动实时 CSI monitor：

```bash
bash start_rx_monitor.sh
```

另一个终端启动 TX：

```bash
bash start_tx.sh
```

monitor 关注指标：

```text
corr max RX0/RX1
sync channel
threshold
frames detected / expected frames
detected rate
extracted frames
CFO mean RX0/RX1
```

### 4.3 关键现象

在 TX 信号较弱时，相关峰偏低，即使降低 threshold，检测到的 frame rate 也明显低于理论值。

增强 TX 后，monitor 显示：

```text
threshold = 0.35 或 0.45
detected frames ≈ 497 / 497.8
detected rate ≈ 998.4 Hz
probe_rate = 1000 Hz
```

这说明 frame detection 已经接近满检。

证据文件：

```text
experiments/02_frame_threshold_detection/evidence/initial_monitor_threshold_005.png
experiments/02_frame_threshold_detection/evidence/threshold_025_low_detection.png
experiments/02_frame_threshold_detection/evidence/threshold_035_low_detection.png
experiments/02_frame_threshold_detection/evidence/threshold_035_stable_detection_after_tx_gain.png
```

### 4.4 从随机 preamble 到 WiFi-like training 的改进

早期 frame 使用随机 BPSK OFDM preamble：

```text
preamble
preamble
TX0 pilot
TX1 pilot
guard
```

这种结构可以工作，但 frame start 可能存在 sample-level 抖动。OFDM 中 FFT window 偏移 1 个 sample 就会引入明显的子载波线性相位变化，因此会导致复数 CSI 在相邻 frame 间看起来变化很大。

改进后使用 WiFi-like training：

```text
STF-like repeated short training
LTF-like repeated long training
TX0 pilot
TX1 pilot
guard
```

改进点：

```text
training 段同时从 TX0/TX1 发射，提高同步相关峰
LTF-like 长训练段用于更稳定的 CFO 和 frame 对齐
TX0/TX1 pilot 仍然分时发送，保留 2x2 MIMO CSI 估计能力
```

### 4.5 实验结论

实验二确认：

```text
frame detection threshold 需要结合 corr max 和 detected rate 选择
在当前增强 TX 条件下，threshold=0.35 或 0.45 都能稳定检测
WiFi-like training 更适合稳定 frame 对齐和 CSI 提取
```

## 5. 实验三：5 秒 CSI 提取与分析

### 5.1 实验目的

实验三用于采集 5 秒双通道 IQ 数据，并离线提取 2x2 MIMO CSI。

目标是验证：

```text
能否从 IQ 中稳定提取 frame
能否得到完整 H.npy
四条 MIMO link 是否都有有效 CSI
CSI 幅度和相位是否具备 sensing 分析价值
```

### 5.2 运行方式

### 5.3 CSI 数据格式

提取得到的 `H.npy` 形状：

```text
H.shape = (4993, 2, 2, 52)
```

含义：

```text
4993 frames
2 RX channels
2 TX channels
52 active subcarriers
```

布局：

```text
H[frame, rx, tx, active_carrier]
```

在 `probe_rate=1000 Hz` 下：

```text
4993 frames ≈ 4.993 seconds
```

说明 5 秒 capture 中 frame 提取基本完整。

### 5.4 幅度分析

四条链路的平均幅度：

```text
RX0<-TX0: -43.58 dB
RX0<-TX1: -48.13 dB
RX1<-TX0: -42.89 dB
RX1<-TX1: -40.45 dB
```

幅度标准差：

```text
RX0<-TX0: 4.07 dB
RX0<-TX1: 5.39 dB
RX1<-TX0: 2.57 dB
RX1<-TX1: 1.90 dB
```

结论：

```text
RX1<-TX1 是最强且最稳定的链路
RX0<-TX1 是最弱且波动最大的链路
四条链路的幅度结构清晰，说明 CSI 提取有效
```

### 5.5 相位分析

相位分析使用 RX 间相对相位：

```text
angle(H[:, 0, tx, :] * conj(H[:, 1, tx, :]))
```

原因是单链路绝对相位容易受到硬件随机初相、CFO/SFO、frame timing offset 影响，而同一接收设备的 RX0/RX1 相对相位可以抵消一部分公共误差。

观察结果：

```text
相对相位存在明显时间结构
TX1 的相对相位变化强于 TX0
相位曲线不是纯随机噪声
```

这说明相对相位可以作为后续 sensing 特征，但需要进一步滤波和去趋势。

### 5.6 相邻 frame 稳定性分析

直接比较相邻 frame 的原始复数 CSI，得到：

```text
complex correlation mean: 0.743
complex correlation median: 0.775
relative complex change mean: 1.128
mean absolute amplitude step: 3.73 dB
median absolute amplitude step: 3.72 dB
mean absolute phase step: 1.18 rad
median absolute phase step: 1.17 rad
```

这说明：

```text
原始复数 H 并不是逐 frame 完全稳定
相邻 frame 的 raw complex H 不能直接视为真实物理信道变化
幅度和相对相位更适合作为 sensing 特征
```

主要原因：

```text
frame start 估计存在 sample-level 抖动
FFT window 偏移会引入子载波线性相位
两台 B210 没有共享时钟，存在残余 CFO/SFO
每个 frame 的公共相位会漂移
```

OFDM 中如果 FFT window 偏移 `n` 个 sample，第 `k` 个子载波会出现：

```text
exp(-j * 2π * k * n / N)
```

当前 `N=64`，高频子载波 `k=26` 时，1 个 sample 的偏移就会引入约：

```text
2π * 26 / 64 ≈ 2.55 rad
```

因此 raw complex CSI 对 timing offset 非常敏感。

### 5.7 实验结论

实验三说明当前链路已经可以稳定提取 2x2 MIMO CSI：

```text
frame 提取接近完整
H.npy 结构正确
四条 MIMO link 都有有效 CSI
幅度特征具备稳定结构
相对相位具备时间结构
raw complex H 仍需 sanitization 后再用于精细相位分析
```

## 6. CSI Sanitization 需求

当前实验已经能提取 CSI，但要用于更可靠的 CSI sensing，还需要进一步做 CSI sanitization。

测得 CSI 可近似表示为：

```text
H_measured[k, t] = H_true[k, t] * exp(j * phi[t]) * exp(j * a[t] * k) + noise
```

其中：

```text
phi[t]: 每帧公共相位偏移
a[t] * k: timing offset / SFO 导致的子载波线性相位
noise: 噪声
```

后续建议：

```text
固定 frame grid，减少每帧独立相关检测带来的 start 抖动
对每帧相位 unwrap
拟合并去除子载波线性相位
去除公共相位
使用 RX0/RX1 相对相位
使用幅度平滑、PCA、Doppler/STFT 等特征
```

## 7. 总体结论

三个实验逐步验证了从 RF 可见性到 frame detection，再到 CSI 提取和分析的完整流程。

阶段性结论：

```text
实验一证明 TX OFDM 子载波能量可被 RX 观测到
实验二证明增强 TX 后 frame detection 可以接近 1000 Hz 满检
实验三证明 5 秒 IQ 数据可以提取出完整 2x2x52 CSI
```

当前系统已经具备继续开展 CSI sensing 的基础。下一步重点不是“能不能提取 CSI”，而是提升 CSI 质量：

```text
更精确的同步
更稳定的 frame grid
CSI phase sanitization
静态/动态场景对比实验
sensing 特征提取与分类/检测算法
```
